PAH Forward Model — Letter Draft (working notes)

Living summary, updated as the analysis notebook progresses. Companion to notebooks/2026-07-01-pah-forward-model-letter.ipynb.

The question this Letter answers: when we measure the mid-infrared "PAH bump" (a dust-grain emission feature) hiding in broadband Spitzer/Herschel data stacked over thousands of galaxies, does the answer change if we stop assuming the shape of the warm-dust continuum underneath it, and instead let the data tell us its shape?

1 · Loading the data done

We loaded three independent stacking runs, each built from a different (non-overlapping) third of the COSMOS2020 galaxy catalog, stacked against Spitzer MIPS 24μm/70μm and Herschel maps out to 500μm. Comparing the three thirds to each other later gives us an honest error bar that accounts for "which galaxies happened to be in the sample," not just measurement noise. As a bonus, each third also uses slightly offset redshift bins, so combining all three gives finer redshift resolution than any one run alone.

2 · Data consistency checks done

  • 292 usable data points across 3 folds × 4 stellar-mass bins × redshift 0.5–5.0.
  • The redshift-bin offsets are real: at z ≥ 3.5 (where we most need extra resolution to pin down the warm-dust slope) the three runs share zero common bin edges — confirmed on the actual executed data, not just the configuration file, which is important because an earlier version of the config would have produced identical bins at high z.
  • Caution — high-mass, high-z bins are thin. Of the 49 points at z ≥ 3.5, 37 are built from fewer than 75 galaxies (massive galaxies are intrinsically rare, and rarer still at early cosmic times within one field). The most massive bin has stacks with as few as 1–3 galaxies at the highest redshifts — those points are essentially placeholders, not measurements, and shouldn't carry weight in the final fit.
  • The 70μm error inflation used to keep the temperature fit honest does not leak into the PAH-fit error budget. We deliberately tell the fitting code to distrust the 70μm point above z=0.8 when fitting the dust temperature (because at those redshifts 70μm starts to trace short-wavelength features rather than the smooth cold-dust glow). We verified this distrust is not accidentally baked into the error bars we use later for the PAH measurement itself — confirmed by checking the raw data, since an accidental leak would silently kill our ability to test the warm-dust slope with 70μm at high redshift, which is the whole point of this Letter.

3 · Smoothed baseline done

Before hunting for the PAH signal we need a clean estimate of the smooth "background" dust glow it sits on top of. Fitting that background bin-by-bin is noisy — a small wobble in fitted temperature becomes a big wobble in the predicted 24μm flux because the far-infrared-to-24μm extrapolation is very steep. Instead we fit one smooth trend (temperature and amplitude vs redshift and stellar mass) across only the best-measured bins, and use it to predict the background everywhere, including the thin high-redshift bins. This step never looks at the 24/70μm data itself, so it cannot manufacture a fake PAH signal — it only cleans up the ruler we measure the signal against. Result: 30–40% less scatter in the predicted background, in every mass bin.

4 · Forward-model fit, warm-dust slope fixed at α=2 done

Under the standard assumption, the PAH forward model is a clean, strong detection: χ²/dof = 4.5, and every mass bin detects the PAH-relative-to-continuum amplitude at 9–14σ. The amplitude rises with stellar mass (slope +0.44 dex/dex) — more massive galaxies show a bigger PAH signature relative to their own warm-dust continuum. The three PAH feature groups (6.2μm, 7.7+8.6μm, 12.7μm) come out in the ratio 1 : 1.5 : 4.5.

Framing note: this amplitude is a PAH-to-continuum ratio, closer to an equivalent width than to PAH-luminosity-over-total-infrared-luminosity. It rising with mass does not contradict this project's earlier finding that LPAH/LIR is close to flat with mass — the warm continuum itself falls faster with mass than the PAH luminosity does, so the ratio to continuum can rise even while the ratio to total IR luminosity stays flat.

5 · Forward-model fit, warm-dust slope free done

Letting the warm-dust slope float (using MIPS 24 and 70μm jointly, with a weak preference for the textbook value of 2 built in) gives α = 2.86 ± 0.06 — steeper than assumed (χ²/dof = 5.5, using both bands; not directly comparable to the α=2 fit's 4.5, which by design uses 24μm only — see caveat below). Confirmed this is a genuine best answer and not the optimizer running into an artificial wall by widening the allowed range from [1,3] all the way to [1,5]: the answer didn't move.

Caveat, fixed during review: an earlier pass of this analysis called the fitting function with an option combination that silently discarded the 70μm data and fit 24μm alone, despite intending a joint fit — a real bug, caught by tracing through the library source before building the figures below. Refitting correctly (genuinely 24+70μm) moved α from 2.93±0.07 to 2.86±0.06 — a small shift, and every conclusion below already reflects the corrected numbers.

The catch: the absolute PAH amplitude is very sensitive to this choice — it roughly doubles to quintuples across the mass bins (top bin: 7 → 34) when α moves from 2 to 2.86. The key result of this section is not the α value itself — it's that the direction of the mass trend survives it (slope +0.44 → +0.69 dex/dex, same sign, a bit steeper). Any absolute PAH/continuum number this pipeline reports must state which α was used — this is a real systematic, not a hypothetical one.

6 · Robustness: fold-ensemble & redshift-cut checks done

Two independent stress tests, because a single "letting α float" result could in principle be propped up by exactly the thin, unreliable high-redshift bins flagged in Section 2.
  1. Redshift cut (the decisive test): restricting to only the closer, best-measured galaxies (z<2.5, which removes essentially every thin high-z bin) still gives α = 2.87 ± 0.08 — statistically identical to using the full sample out to z=5 (2.86 ± 0.06). The steeper slope is not an artifact of the sparse high-redshift tail.
  2. Fold-ensemble: refitting each of the three independent galaxy samples separately (rather than trusting one pooled statistical error) gives α = 3.17, 3.27, 2.40 — averaging 2.95 ± 0.28. All three independently prefer a steeper slope than 2. (First attempt at this check hit an artificial ceiling built into the fitting code's default settings for two of the three samples, which looked like disagreement between samples but was really the knob being turned too low — corrected by widening it, which is why this took two passes.)
A third check: the fast baseline-rescaling shortcut used above skips a secondary temperature-vs-redshift term for speed. Rebuilding the baseline exactly instead pushes the best-fit slope further from 2 (3.26 instead of 2.86), not back toward it — the shortcut was, if anything, conservative. But the fast (2.86) and exact (3.26) treatments land at essentially identical fit quality (χ²/dof 5.50 vs 5.51) — the data can't distinguish between the two baseline shapes, so α partly slides to absorb which baseline construction we impose. That 0.4 spread between methods is the dominant systematic on α, larger than the ±0.28 fold-to-fold statistical scatter — the two should not be read as independent small errors.

Bottom line: the honest claim is "α is robustly steeper than 2" — direction bulletproof, exact value method-limited (2.9–3.3, with the baseline-construction choice as the leading uncertainty). It is not an artifact of sparse data or a tuning knob. The mass trend is the headline result, and this section's job was to show it survives the α question — checked now at α=2, 2.86, and 3.26, positive every time.

7 · Literature context

  • Mass trend: not in tension with the classic "PAH deficit" literature (Smith et al. 2007, SINGS IRS; Galliano et al. 2021, DustPedia) — those measure a different ratio (PAH luminosity fraction, not PAH-to-warm-continuum), and this project's own prior result found LPAH/LIR flat with mass while this continuum-normalised amplitude rises.
  • Physical mechanism (if the mass trend survives further scrutiny): radiation-field intensity, not halo mass — Egorov et al. (2025) and Leroy et al. (2023, PHANGS-JWST) show PAH fraction anti-correlated with the UV ionization parameter on resolved HII-region scales. Tielens (2008) is the physics reference for why the 6.2/7.7+8.6/12.7 band ratios trace grain ionization and size.
  • Warm-dust slope α=2.9: no direct literature value to compare against — this is closer to a pipeline-methodology result than an astrophysical one, but it directly affects any absolute PAH/continuum number derived from this code, including the Tdust(z) correction motivated by Viero et al. (2022) and Schreiber et al. (2018).
  • Independent cross-checks worth pursuing: arXiv:2606.20809 (2026 PAH lifecycle cosmological simulations) for a theoretical α(M*, z) prediction; arXiv:2606.18244 (PAHSPECS, JWST MIRI MRS at z~1–3) for direct spectroscopy that needs no continuum assumption at all.

8 · The model against the data done

Two figures, both at the best-fit α=2.86, one panel per stellar-mass bin (see the notebook for the full-resolution versions):
  • Model overlay: raw stacked 24μm flux vs redshift, with the smooth continuum-only curve and the continuum+PAH curve both drawn on top. The full model visibly tracks wiggles in the data — most clearly a bump around z≈1.5–2.2 — that the smooth continuum alone misses.
  • Deconvolved spectrum, coloured by redshift: the same fit re-expressed as PAH excess over continuum vs rest-frame wavelength (the quantity this whole method exists to recover), with points coloured by which of four redshift slices they come from. The classic PAH bump (rest-frame ∼6–9μm) is traced out consistently where different-coloured (different redshift) points overlap in wavelength — the real test that this is a genuine spectral feature sweeping through the bandpass with redshift, not a coincidence tied to one epoch. The highest-mass panel visibly shows the large scatter/error bars in the green/red (z>2.5) points flagged as thin in Section 2 — consistent with, and a direct visual confirmation of, that earlier caveat.
  • The underlying PAH lines, pre-convolution, by mass bin and redshift slice: strip away the bandpass and redshift bookkeeping entirely, and plot the intrinsic PAH line template itself — narrow peaks at the real rest wavelengths (6.2, 7.7, 8.6, 12.7μm). One panel per stellar-mass bin; within each, four curves for four redshift slices, from the sSFR-anchored evolving fit. Top row (absolute amplitude): the low-redshift curve is dramatically taller than the others in every panel — this is a known runaway (amplitude evolution is scatter-limited on this dataset, matching a previously-documented artifact almost exactly: ηamp≈−2.39 here vs −2.37 previously), not a physical detection. Bottom row (each curve peak-normalised, dividing the runaway out): isolates the line-ratio evolution, which is comparatively better-behaved and shows a consistent pattern across all four mass bins — the 6.2μm feature relatively dominant at low z, the 12.7μm feature relatively dominant at high z, with 7.7+8.6μm in between. Interesting and consistent across mass bins, but still exploratory (built on the same fit machinery whose amplitude term is known to be scatter-limited) — a hint worth following up, not a claimed result.

9 · Putting the correction into the SED fit itself done

Everything above lives in the notebook's own machinery. This section pushes the measured PAH model into the shared simstack4 library (greybody.py) so the actual SED-fitting code — the same code every other branch of this project calls — includes it, using the best-fit Wien slope (α=2.86) instead of the textbook 2, and a freshly derived PAH-amplitude relation. Two additions to greybody.py made this possible, both additive (nothing existing changed behavior):
  • alpha_wien is now an overridable attribute (was hardcoded at 2.0 inside greybody_model's default argument). Every pre-existing call site omits it and is byte-for-byte unaffected.
  • A new wien_mode="lir_pah" path (_pah_flux_lir) predicts the PAH flux from a freshly measured log10(LPAH/LIR) = a·logM* + d relation — using the properly 8–1000μm-integrated LIR (calculate_LIR), not the discredited f24/fpeak proxy this project deliberately moved away from — converted to an in-band flux via the real MIPS bandpass kernel (feature_band_curves).
Derived relation (at αwien=2.86): a fresh fit across the four mass bins gives log10(LPAH/LIR) = a·logM* + d, with LPAH/LIR in the few-percent range and rising with mass — consistent in sign and magnitude with the earlier pah-signflip-diagnosis finding that this continuum-normalized amplitude behaves oppositely to the classic literature PAH-deficit ratio.
Round-trip validation — two real bugs found and fixed along the way, both worth recording because either would have silently produced a wrong SED plot:
  1. Amplitude placeholder. An early version of the check passed a dummy amplitude=0.0 into _pah_flux_lir instead of each point's real fitted greybody amplitude. Since the model uses A=10amplitude, amplitude=0.0 silently substitutes A=1 for the true A≈10−35 — an astronomical (1037%) disagreement that was impossible to miss.
  2. alpha_wien mismatch (subtler, harder to catch). After fixing (1), the check still disagreed by a consistent ~18–21% at every point, independent of redshift or mass bin — the flat, uniform size of the residual (not the physically-expected pattern of point-to-point scatter) was the tell that something was still structurally wrong, not just noisy. Root cause: the notebook's own independent re-derivation of LIR (used both to fit the (a,d) coefficients and to cross-check the library call) used a freshly constructed Greybody() instance that still had the default alpha_wien=2.0, while the library call it was being checked against used the fitted 2.86. LIR integrates through the Wien-side power law, so this mismatch biased the whole derived relation by a near-constant factor. Fixed by setting alpha_wien consistently on both instances before deriving (a,d) — after which the round-trip check passes at exactly 0.0000% disagreement (16/16 points, machine precision).
A third, non-bug finding surfaced by the same debugging: the very first version of this check compared the library's sharp-redshift evaluation against the dither-tomography measurement's bin-integrated kernel (which folds in the stacking bin width, dN/dz, and photo-z smearing) and found 48–100% disagreement, worst at high z. Tracing it down (rather than loosening the tolerance) showed the two kernels track at z≲1.5 and diverge sharply above that — not from a bug, but because the dither fit used a small (3%) catastrophic-outlier redshift pedestal (f_cat=0.03) that leaks low-z-consistent PAH flux into nominally high-z bins where the true, sharp-z response is genuinely ~0 (the feature has swept out of the MIPS 24μm band by z∼3.5–4). The library's sharp-z evaluation is the physically correct choice for injecting into a per-population SED fit (matching how every other quantity in greybody_model is evaluated), so the round-trip check was rebuilt to test the library's own documented formula against an independent notebook-side computation of that same formula, rather than against the definitionally-different dither-tomography estimate.
Two-pass refit result. Refit with wien_mode="lir_pah", alpha_wien=2.86, the fresh (a,d) coefficients, and 24μm inflation made redshift-dependent (4×, i.e. included, at z<3 where the PAH correction is active and validated; 10000×, i.e. excluded, above z=3 where the feature has swept out of the band and there is nothing to correct against — mirroring how 70μm is already split at z=0.8).
  • Tier counts are unchanged (Tier A/B/C: 83/95/102 before and after). Tier is set by the median signal-to-noise across all 8 bands, and 24μm — even once it is allowed to matter — is not usually the band that determines that median, so this correction alone doesn't flip Tier C bins to Tier B on this dataset.
  • Dust temperature does shift: +0.81 K on average at z<3 (where the correction is active), vs. a smaller +0.43 K at z≥3 (where the PAH term is off, and the residual shift comes entirely from using the measured αwien=2.86 instead of 2 for the general Wien-side continuum slope — a separate, already-validated Section 5–7 result, not the PAH correction itself). The z<3 shift being roughly double the z≥3 one is consistent with the PAH correction adding a real, modest effect on top of that baseline retilt.
The resulting SED grid (96 panels, redshift × stellar-mass) now shows 24μm as a real, model-informed data point at z<3 instead of an excluded placeholder.

10 · Headline pivot: the PAH deficit rises with mass decided 2026-07-01

Section 9's fresh log10(LPAH/LIR) = a·logM* + d fit, done at the branch's own best-fit αwien=2.86 (not the old assumed 2), gives 5.9% / 8.4% / 8.7% / 11.5% across the four mass bins — a clear rise (+0.212 dex/dex), not the flat ≈2.4% found in the 2026-06-28 analysis. That earlier result is this project's previous Letter framing ("PAH tracks LIR, invariant of stellar mass") — directly contradicted by this number.

Before trusting a number this different from the established framing, we ran the obvious objection as an explicit check rather than assuming it away: this pipeline's own PAH amplitude is already known to be strongly sensitive to α (a 0.5 change in α moves the raw amplitude 3–4×) — is the "rise" just that systematic in disguise? Re-running the identical calculation with α pinned at the old assumed value (2.0) instead of the fitted 2.86, on the exact same data and baseline, gives +0.146 dex/dex — still clearly rising, same sign, only 1.5× different in size. That is a much smaller swing than the amplitude's own 3–4× sensitivity, so the rising trend is not an artifact of which α we assume.

It is not, however, a clean resolution of the original tension: at that same α=2, this session's result (+0.146, rising) still disagrees with the original 2026-06-28 measurement (+0.019, flat) made at the identical assumed slope. That earlier number used a single catalog split and a less careful (unsmoothed) baseline construction; this session pools three independent catalog splits and smooths the baseline first. Which of those differences is actually responsible is not yet isolated — an open item for the next phase of work.

Decision (2026-07-01): given the rise is robust to the α question but not yet fully reconciled with the older measurement, the Letter's headline claim pivots to "the PAH deficit rises with stellar mass" — supported by two independent lines of evidence (this new luminosity-ratio measurement and the equivalent-width-style ratio from Sections 5–7, both rising and both checked against the α systematic) — retiring the "PAH tracks LIR, invariant of mass" framing.
